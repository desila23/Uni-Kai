# Caratteristiche di Java

### Funzione vs Metodo
- Una **funzione** è indipendente, mentre un **metodo** è sempre associato a un oggetto o a una classe.

---

## Stringhe
- Le stringhe fanno parte del linguaggio di base (classe `String` del package `java.lang`, importata automaticamente).  

---

## Librerie
- In Java la maggior parte delle classi utili è già inclusa (nel package `java.lang`).  
- Per altre funzionalità (collezioni, I/O, rete, ecc.) bisogna importare le librerie tramite `import`.  
- Ciò che importiamo sono **classi**, con i loro metodi e attributi.  

Esempio (riprendendo dal `HelloWorld`):  
```java
System.out.println("TESTO");
```
`println` è un **metodo** della classe `PrintStream`, accessibile tramite `System.out`.

---

## Classpath
- Quando compilo un file `ES1.java`, `javac` genera `ES1.class`.  
- Questo `.class` viene caricato dalla JVM seguendo il **classpath**, ovvero il percorso in cui si cercano le classi compilate.  
- Possiamo modificare il classpath con l’opzione `-cp`.

## Oggetti
- Un oggetto ha uno **stato** (valori delle variabili istanza) che può cambiare durante l’esecuzione.  
- Gli oggetti sono istanze di una **classe**.  

Esempio:
```java
class Scrivania {
    float altezza;
    float costo;
    String colore;

    void printDati() {
        System.out.println("Altezza = " + altezza + " metri" +
                           "\nCosto = " + costo + " euro" +
                           "\nColore = " + colore);
    }
}

class Test {
    public static void main(String[] args) {
        Scrivania sandoz = new Scrivania();
        sandoz.altezza = 2;
        sandoz.costo = 5;
        sandoz.colore = "Marrone";

        sandoz.printDati();
    }
}
```

>[!info]- Creazione di un oggetto in java
> Creiamo uno spazio in memoria dinamico (heap) e salviamo il suo riferimento con un “puntatore”, questo puntatore sarà ad esempio `nomeclasse nomeoggetto` e il suo riferimento sarà = `new nomeclasse()

>[!question]- Se faccio una uguaglianza tra due puntatori ad oggetti? `nomeoggetto1=nomeoggetto2`
> Avrò lo stesso puntamento a memoria quindi entrambe puntano allo stesso oggetto

## Ereditarietà
- Una sottoclasse eredita tutte le caratteristiche della superclasse (o classe madre) e può aggiungerne di nuove.  
- In Java non esiste **ereditarietà multipla** tra classi.  
- Con `@Override` si ridefinisce un metodo della superclasse.  

Esempio:
```java
class Animale {
    int peso;
    int anni;
    String nome;

    void printDati() {
        System.out.println("Nome: " + nome + ", Anni: " + anni + ", Peso: " + peso);
    }
}

class Cane extends Animale {
    String razza;

    @Override
    void printDati() {
        super.printDati();
        System.out.println("Razza: " + razza);
    }
}


/* Spiegazione codice
	Ho creato una classe "animale" in cui ho inserito 
	   - le caratteristiche principali di quell'animale
	   - un metodo che mi stampa questi dati
	
	Ho creato poi una classe "Cane" più specifica che ESTENDE la classe "Animale",     questo vuol dire che la classe "Cane" contiene tutte le caratteristiche della
	classe Animale e posso aggiungere altro.
	
	Con "@override" ho fatto si che il metodo "printdati" potesse essere 
	aggiornato, infatti:
	   - con "super." ho inserito il "printdati" di "Animale" all'interno del
	     "printdati" di "Cane" (così posso stampare le caratteristiche di Animale)
       - e poi ho aggiunto una nuova cosa da stampare 
	
	All'interno del main ho creato un "cane" e come vedi ho inserito tutte le
	caratteristiche presenti in "Animale" e le caratteristiche (solo razza in
	questo caso) presenti in "Cane".
	Come vedi ho inserito solo un ".printdati" perché ora è aggiornato.
```

## Interfacce
- Un’interfaccia definisce un **contratto**: un insieme di metodi che una classe DEVE implementare.  
- Le classi usano `implements` per adottare un’interfaccia.  
- Posso implementare più interfacce contemporaneamente.  

Esempio:
```java
interface Animale {
    void verso();
}

class Cane implements Animale {
    @Override
    public void verso() {
        System.out.println("Bau bau");
    }
}

class Gatto implements Animale {
    @Override
    public void verso() {
        System.out.println("Miao miao");
    }
}
```

Altro esempio (ereditarietà multipla simulata con interfacce):
```java
interface Terra { void guida(); }
interface Acqua { void naviga(); }

class Hovercraft implements Terra, Acqua {
    int vel;
    @Override 
    public void guida() { 
	    System.out.println("Guida a " + vel + " km/h"); 
	}
	
    @Override 
    public void naviga() { 
	    System.out.println("Naviga a " + vel + " km/h"); 
	}
}
```


>[!question]- A che servono quindi le interfacce? 
>Le interfacce servono per **fornirci un vocabolario unificante per permettere l’interoperabilità tra oggetti.**
>Interoperabilità indica la capacità di essere flessibile e integrare più linguaggi di programmazione o servizi esterni.
>Java non è un ecosistema chiuso.



#### Esempio finale interfaccia
```java
interface TERRA{
    void GUIDA();
}

interface ACQUA{
    void NAVIGA();
}


class AUTOMOBILE implements TERRA{
    int vel;    
    @Override
    public void GUIDA(){
        System.out.println("La tua auto va a una velocità di " + vel + "km/h");
    }
}

class BARCA implements ACQUA{
    int vel;
    @Override
    public void NAVIGA(){
        System.out.println("La tua barca nagiva a una velocità di " + vel +                                   "km/h");
    }
}

class HOVERCRAFT implements TERRA, ACQUA{ // posso unire due interfacce con ","
    int vel;
    @Override
    public void GUIDA(){

    } 
    
    @Override
    public void NAVIGA(){
        System.out.println("Il tuo hovercrafft nagiva a una velocità di " + vel +                             "km/h");
    }
}


class interfacce{
```

## Package
- Un **package** è come una cartella che contiene classi correlate.  
- Serve a organizzare il codice e ad evitare conflitti di nomi.  

Esempio:
```java
package animali;

public class Cane {
    public void abbaia() {
        System.out.println("Bau Bau!");
    }
}
```

Per usarlo:
```java
import animali.Cane;

public class Main {
    public static void main(String[] args) {
        Cane mioCane = new Cane();
        mioCane.abbaia();
    }
}
```
Con `import animali.*;` si importano tutte le classi del package.

## Variabili in Java
Tipologie:  
1. **Variabili globali** (in Java: campi `static`).  
2. **Variabili di istanza**: descrivono lo stato di un oggetto.  
3. **Variabili locali**: dichiarate dentro un metodo.  
4. **Parametri**: variabili ricevute da un metodo.  

👉 Differenza:  
- **Parametro**: variabile formale dichiarata nel metodo.  
- **Argomento**: valore passato quando il metodo viene chiamato.  

---

## Static vs Non static
- **Non statico**: ogni oggetto ha il proprio valore della variabile.  
	Se modifico la variabile in un oggetto, non cambia nelle altre istanze.
```java
class Persona {
    String nome; // variabile non statica
}

public class Main {
    public static void main(String[] args) {
        Persona p1 = new Persona();
        p1.nome = "Mario";

        Persona p2 = new Persona();
        p2.nome = "Luigi";

        System.out.println(p1.nome); // Mario
        System.out.println(p2.nome); // Luigi
    }
}
```
- Qui **ogni oggetto ha il proprio "nome"**.


- **Statico**: variabile condivisa tra tutte le istanze.  
	Appartiene alla classe, non al singolo oggetto.
```java
class Contatore {
    static int count = 0; // variabile statica
    Contatore() {
        count++;
    }
}

public class Main {
    public static void main(String[] args) {
        new Contatore();
        new Contatore();
        new Contatore();
        System.out.println(Contatore.count); // 3
    }
}
```
- Qui il `count` è **unico** e vale 3 indipendentemente da quanti oggetti creo.


---



```java
class Libro {
    String titolo;        // variabile di istanza (va nell'Heap)
    static int totale = 0; // variabile statica (non va nell'Heap)
    
    Libro(String t) {
        titolo = t;
        totale++;
    }
}

public class Main {
    public static void main(String[] args) {
        Libro l1 = new Libro("Java Base");   // oggetto in Heap
        Libro l2 = new Libro("Java Avanzato"); // altro oggetto in Heap

        System.out.println(l1.titolo); // Java Base
        System.out.println(l2.titolo); // Java Avanzato
        System.out.println(Libro.totale); // 2 (unica variabile condivisa)
    }
}
```
- `l1` e `l2` sono **oggetti distinti nell’Heap**.
- `titolo` è diverso per ogni oggetto.
- `totale` è **unico per la classe** e non vive nell’Heap.
