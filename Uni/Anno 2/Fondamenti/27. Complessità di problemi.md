Finora la teoria della complessità è stata studiata sui **linguaggi**, cioè insiemi di stringhe.  
Ora l’obiettivo è estendere questi concetti ai **problemi decisionali**.

Un problema, però, non è direttamente un linguaggio.  
Per poter parlare di complessità, è necessario **trasformare un problema in un linguaggio**.

Questa trasformazione avviene tramite una **codifica** delle istanze del problema in stringhe.  
Affinché la complessità abbia senso, la codifica deve essere **ragionevole**, cioè non deve alterare artificialmente la dimensione dell’input.


## Da problema a linguaggio
Sia $\Gamma$ un problema decisionale descritto dalla tripla $$\Gamma = \langle I_{\Gamma}, S_{\Gamma}, \pi_{\Gamma} \rangle$$
L'insieme delle istanze $I_{\Gamma}$ si divide in: 
- **istanze sì**: istanze che soddisfano il predicato $\pi_{\Gamma}$
- **istanze no**: istanze che non lo soddisfano

Introduciamo una codifica $$χ : I_{\Gamma} → Σ^{*}$$che associa a ogni istanza una stringa.

Questa codifica induce una partizione di $Σ^{*}$ in tre insiemi:
1. stringhe che codificano istanze sì
2. stringhe che codificano istanze no
3. stringhe che non codificano alcuna istanza del problema

Il **linguaggio associato al problema Γ**, tramite la codifica χ, è l’insieme delle stringhe che codificano **istanze sì**.

Formalmente:
![[Pasted image 20251219164651.png|center]]
Quindi il linguaggio contiene esattamente le codifiche delle istanze per cui la risposta al problema è “sì”.


Dire che un’istanza y è una istanza sì del problema Γ equivale a dire che la stringa x = χ(y) appartiene al linguaggio $L_{Γ}(χ)$.

Viceversa, per decidere se una stringa x appartiene a LΓ(χ) è necessario svolgere due verifiche:
1. verificare se x è effettivamente la codifica di una istanza y del problema
2. se lo è, verificare se l’istanza y soddisfa il predicato $π_{Γ}$

>[!tip] Questo punto è fondamentale:  
La complessità di un problema decisionale non dipende SOLO dalla difficoltà di verificare il predicato, ma anche dalla difficoltà di riconoscere se una stringa rappresenta una **istanza valida** del problema.
>
Questo aspetto distinguerà lo studio della complessità dei problemi da quello dei linguaggi.


## Complessità di un problema
A questo punto si può definire formalmente la **complessità computazionale di un problema decisionale**.

Sia $Γ = ⟨ I_{Γ}, S_{Γ}, π_{Γ} ⟩$ un problema decisionale.  

Sia inoltre **C** una classe di complessità, ad esempio:
- `DTIME(f(n))`
- `DSPACE(f(n))`
- `NTIME(f(n))`
- `NSPACE(f(n))`    
con `f` funzione totale e calcolabile.

Si dice che **Γ appartiene alla classe C** se **esiste una codifica ragionevole**  
$χ : I_{Γ} → Σ^{*}$  
tale che il linguaggio associato al problema, $L_{Γ}(χ)$, appartiene alla classe C.

In simboli:  
$$Γ ∈ C ⇔ \text{esiste una codifica ragionevole χ tale che } L_{Γ}(χ) ∈ C$$

>[!tip] Questa definizione chiarisce un punto importante:  
la complessità di un problema decisionale viene definita **tramite la complessità del linguaggio** che rappresenta le sue istanze sì.

Di conseguenza, per decidere se una stringa x appartiene a $L_{Γ}(χ)$, bisogna:
1. verificare se x è la codifica di una istanza del problema
2. verificare se l’istanza soddisfa il predicato πΓ
Entrambe le operazioni contribuiscono al costo complessivo.

## Il linguaggio delle istanze
Dato un problema decisionale Γ e una codifica ragionevole  $$χ : I_{Γ} → Σ^{*}$$si definisce il **linguaggio delle istanze** di Γ come: $$χ(I_{Γ}) = { x ∈ Σ^{*} : \text{esiste } y ∈ I_{Γ} \text{ tale che } x = χ(y) }$$
Questo linguaggio contiene **tutte e sole le stringhe che rappresentano istanze valide del problema**, indipendentemente dal fatto che siano istanze sì o istanze no.

Poiché χ è una codifica:
- istanze diverse hanno codifiche diverse
    - tipo se $y, z \in I_{\Gamma}$ -> le loro codifiche saranno diverse
- quindi χ è una funzione **invertibile** sul suo dominio
	- nel senso che possiamo risalire all'istanza originale di ogni codifica

Questa verifica è esattamente il problema **Hamiltonian Cycle**.
