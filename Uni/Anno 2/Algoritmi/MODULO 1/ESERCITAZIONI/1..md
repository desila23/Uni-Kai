![[Pasted image 20250606104046.png]]L'idea è quella di inizializzare due variabili
- `i = 0`
- `j = n`

1. Inizialmente osservo se `A[i] > x`, se fosse vero (dato che l'array è ordinato) non potrò mai trovare due valori che sommati restituiscano `x` -> ritorno `-1, -1`.
2. Una volta appurato che `i` può andare bene, faccio la stessa cosa con `j`, ma se mi accorgo che `A[j] > x`, decremento `j` finché non arrivo ad un numero che può andare bene.
3. Inizio a sommare i vari numeri
	- se il valore che ottengo è maggiore di `x`, il problema è causato da `j` -> lo decremento e sommo di nuovo
	- se il valore che ottengo è minore di `x`, il problema è causato da `i` -> lo incremento e sommo di nuovo
	- se il valore che ottengo è esattamente `x` -> ritorno `i` e `j`
4. Se non ottengo mai `x`, allora non esistono i due valori -> ritorno `-1, -1`

PSEUDOCODICE
```c
Ricerca numeri(x)
i = 0
j = n

while (i < j) do                     // O(n)
	if A[i] + A[j] = x then
		return (i, j)
	
	else if A[i] + A[j] > x then
		j--
	
	else i++

return (i = -1, j = -1)


// COSTO TOTALE nel caso peggiore: O(n)
```
