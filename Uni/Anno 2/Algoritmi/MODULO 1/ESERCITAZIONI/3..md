#### Pseudocodice
```c
VisitaBFS_Cammmino_rosso(vertice s)
	Coda C
	C.enqueue(r)
	
	max_sum
	
	while(not C.isempty()) do
		u <- C.dequeue()
		if(u != null && u == R) then
			val(u) = val(u) + val(p(v))
			
			if((val(u)) > max_sum) then
				max_sum = val(u)
			
			C.enqueue(sx(u))
			C.enqueue(dx(u))
	
	return max_sum
```


#### Versione ricorsiva Guala
```c
MaxRosso(vertice s)
	if s = NULL then return 0   // se sono arrivato ai figli delle foglie
	if col(s) = then N return 0
	
	return val(s) + max{MaxRosso(sx(s)), MaxRosso(dx(s))}
```

#### Costo
Dato che ogni nodo viene visitato al più una volta, il costo è $$O(n)$$

#### Correttezza
Si richiedeva di calcolare il valore del cammino rosso massimo.
Nella coda vengono inseriti tutti i nodi ma poi vengono analizzati solo quelli di colore "Rosso".
La visita BFS fa sì che tutti i nodi vengano visitati per livello, così risulta più facile calcolare singolarmente il valore del cammino radice-nodo.
Dato che il valore aggiornato di ogni nodo è dato dal suo valore originale sommato al valore del padre, e questa cosa vale anche per il padre e per gli antenati del nodo, ogni volta in un nodo `u` ci sarà il valore totale del cammino nodo-radice.
Questo fa sì che il cammino venga generato "indirettamente".

Ogni volta viene confrontato il valore del cammino sul nodo attuale con il massimo "precedente" e se il cammino attuale è > allora diventerà il nuovo massimo.

# ESERCIZIO 2
![[Pasted image 20250608104405.png|center|500]]
L'idea è fare una visita BFS (che mi ritorna i cammini minimi dalla radice) e poi prendere solo quelli che hanno distanza $\ge$ h

#### PSEUDOCODICE
```c
Profondita(s, h)
	Coda C
	P.enqueue(s)
	dist(s) <- 0
	aggiungo per ogni nodo il campo dist(v) = NULL
	
	contatore_profondita <- 0
	
	while(not C.isempty()) do
		u <- C.dequeue()
		if(u != NULL) then
			if(dist(u) == NULL) then
				dist(u) <- 1 + dist(padre(u))
			
			if(dist(u) >= h) then 
				contatore++
			
			C.enqueue(sx(u))
			C.enqueue(dx(u))
	
	return contatore_profondita
```

Versione ricorsiva
```c
Prodonfita(v, h, i)
	if (v == NULL) then return 0
	if (i >= h) then
		return 1 + Profondita(sx(v), h, i+1) + Profondita(dx(v), h, i+1)
	else return Profondita(sx(v), h, i+1) + Profondita(dx(v), h, i+1)
```

#### Costo
I nodi vengono visitati al più una volta, quindi $$O(n)$$

#### Correttezza
Deriva dalla correttezza della visita BFS, con l'unica aggiunta di un contatore per per analizzare se la profondità di un nodo `v` sia maggiore o uguale dell'`h` data in input.
