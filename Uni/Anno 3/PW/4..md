## Git Branch
Linee di sviluppo parallele e indipendenti

Generalmente
- abbiamo una linea di sviluppo principale (`main`)
- le linee secondarie servono per modificare il main
	- poi facciamo il merge

### Idea
L'idea è fare tot branch e tot commit (che poi non è così però boh)

### Sintassi
NUOVO BRANCH
```
git branch <nome>
```

CAMBIARE BRANCH
```
git checkout <nome/id>
```
- cambia la WORKING DIRECTORY

CAPIRE DOVE STO LAVORANDO
```
HEAD
```

SALVATAGGIO TEMPORANEO
```
git stash
```

RECUPERO DELLO STATO SALVATO DA STASH
```
git pop
```

MERGE
- fast farward -> faccio proseguire il main sui commit dell'altro 
	- tipicamente lo faccio se non ho troppe modifiche (?)
- Merge commit -> quando è già stato fatto un commit sul main ma io ho fatto un branch su una parte di codice PRECEDENTE, quindi il nuovo codice ha delle parti che il vecchio non aveva

	- se i due commit sono in conflitto tra di loro bisogna scegliere quale tenere (lo scelgo io)
	- lavora riga per riga (?)
- potrei anche avere casi in cui anche se è possibile fare il fast farward MI OBBLIGHINO a fare il merge
