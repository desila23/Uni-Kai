Alcune volte siamo root ma non nell'ambiente principale.
In ambienti distribuiti si fanno dei container
- molto spesso siamo root in un container

### Come identificare se sono root in un container

### Container
Un'unità software che contiene codice e tutte le sue dipendenze (librerie, file di configurazioni, ecc.).

##### Differenza tra VM e Container
- container -> viene condiviso (virtualizzato credo) il SO
- VM -> viene virtualizzato l'HW


### Isolamento dei containers
Puoi farle tramite kernell
1. NAMESPACE -> assicurano che un processo in un container non interferisca con quello di un altro
2. CONTROL GRUP -> gestiscono quante risorse può utilizzare un determinato container
3. Capabilities -> quando un container viene inizializzato, le capabilities più pericolose vengono tolte


#### Namespaces

## Docker (container engine più famoso)
Architettura client server e il client comunica tramite un DAEMON.

#### Processo di creazione di un container
- Inizia con un docker file
	- dice come è fatto sulla carta il nostro container 
		- su ogni layer ho un comando, ecc...
- viene generata una docker image
- questa docker image viene runnata e viene generato il container
	- viene aggiunto un layer aggiuntivo che è l'unico scrivibile
	- quando distruggo il container viene distrutto il layer ma l'immagine rimane
