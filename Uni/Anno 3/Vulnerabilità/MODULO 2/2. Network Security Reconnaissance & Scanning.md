## 1. DNS Reconnaissance (Avanzato)
Una volta identificato il dominio target, l'obiettivo è mappare tutti i sottodomini e gli host associati.

### A. Forward Lookup Bruteforce
Si tenta di indovinare i nomi degli host (es. `dev`, `vpn`, `test`) usando una wordlist.
```bash
# Esempio Bash script rapido (da terminale)
for sub in www ftp mail dev vpn; do host $sub.target.com; done | grep "has address"
```

### B. Reverse Lookup Bruteforce
Si scansiona un range di IP cercando i record **PTR** (chi è questo IP?).
```bash
# Esempio: scansione del range .1-.255 per trovare nomi host
for ip in $(seq 1 255); do host 38.100.193.$ip; done | grep -v "not found"
```

### C. DNS Zone Transfer (AXFR)
È il "colpo grosso": se il server è malconfigurato, ti invia l'intero database della zona.
```bash
# Tentativo con host
host -l <domain> <nameserver>

# Tentativo con dig
dig axfr @<nameserver> <domain>
```

### Tool Dedicati (Automazione)
*   **DNSRecon**: Molto completo.
    ```bash
    dnsrecon.py -d <domain> -t axfr        # Zone Transfer
    dnsrecon.py -r <startIP>-<endIP>      # Reverse Lookup
    dnsrecon.py -d <domain> -D <list> -t brt # Bruteforce
    ```
*   **DNSEnum**: Esegue tutto in automatico (scansione, bruteforce, whois).
    ```bash
    dnsenum <domain>
    ```
*   **Fierce**: Ottimo per trovare spazi IP non contigui.
    ```bash
    fierce --domain <domain>
    ```

## 2. Host Discovery (Individuare macchine accese)
Prima di scansionare le porte, dobbiamo sapere quali macchine rispondono.

*   **ARP Scan (Stessa sottorete)**: È il più affidabile perché non può essere bloccato dai firewall software.
    ```bash
    nmap -sn -PR <network/24>
    ```
*   **Ping Scan (Reti diverse)**: Invia ICMP Echo Request. Spesso bloccato dai firewall.
    ```bash
    nmap -sn -PE <target_network>
    ```

## 3. Port Scanning: Meccanismi e Tecniche
Il port scanning serve a capire se una porta è `OPEN`, `CLOSED` o `FILTERED` (firewall).

### TCP Three-Way Handshake (Recap)
1.  **SYN** (Client → Server)
2.  **SYN/ACK** (Server → Client) → *Porta Aperta*
3.  **ACK** (Client → Server)
*   Se il server risponde **RST**, la porta è **chiusa**.

### Tipologie di Scan Nmap (Cheat Sheet Comandi)

| Comando | Nome | Logica | Pro/Contro |
| :--- | :--- | :--- | :--- |
| `nmap -sT` | **TCP Connect** | Completa l'handshake (SYN-SYN/ACK-ACK). | **Rumoroso**: appare nei log applicativi. |
| `nmap -sS` | **SYN Scan** | Invia SYN, aspetta SYN/ACK, poi invia RST. | **Stealth**: non apre la sessione. Richiede root. |
| `nmap -sU` | **UDP Scan** | Invia pacchetto UDP vuoto. | **Lento e inaffidabile**: se riceve ICMP Unreachable = Chiusa. |
| `nmap -sF` | **FIN Scan** | Invia FIN (chiusura). | Bypass firewall semplici. Non funziona su Windows. |
| `nmap -sX` | **Xmas Scan** | Invia FIN, PUSH e URG ("acceso come un albero"). | Come FIN, induce risposte diverse in base all'OS. |
| `nmap -sN` | **NULL Scan** | Nessun flag impostato. | Come sopra. |
| `nmap -sA` | **ACK Scan** | Invia solo ACK. | **Non trova porte aperte**. Serve a mappare le regole del firewall. |

### Opzioni Strategiche di Nmap
*   **Timing (`-T0` a `-T5`)**:
    *   `-T0` (Paranoid) / `-T1` (Sneaky): Lentissimi, per eludere IDS.
    *   `-T4` (Aggressive): Standard per CTF/Esami.
    *   `-T5` (Insane): Velocissimo, rischia di perdere pacchetti.
*   **Evasione Firewall**:
    *   `-P0`: Non pinga l'host (utile se l'host blocca ICMP).
    *   `-D <IP1,IP2,ME>`: **Decoy**. Camuffa il tuo IP tra altri IP falsi nei log del target.
    *   `-S <IP_falso>`: Spoofing dell'IP (ma non riceverai le risposte).

## 4. Analisi dei Servizi e OS Fingerprinting

### Banner Grabbing (Identificare Versioni)
Serve a capire *esattamente* quale software gira su una porta (es. Apache 2.4.1).
```bash
# Con Netcat
nc -v <target> <port>

# Con Nmap (Version Detection)
nmap -sV <target>
```

### OS Fingerprinting (`-O`)
Nmap invia pacchetti TCP/UDP "strani" e analizza come l'OS risponde (ogni sistema ha piccole varianti nello stack TCP).
```bash
nmap -O <target>
```
>[!warning] Accuratezza
>L'OS detection è molto più affidabile se Nmap trova almeno **una porta aperta** e **una porta chiusa**.

## 5. Nmap Scripting Engine (NSE)
Script in Lua per automatizzare compiti complessi. Gli script si trovano in `/usr/share/nmap/scripts`.

*   **Default scripts**: `nmap -sC <target>`
*   **Specifico script**: `nmap --script=http-title <target>`
*   **Categorie**: `nmap --script=safe|intrusive|vuln <target>`

## 6. Enumerazione Servizi Specifici

### SMB (Server Message Block - Porte 139, 445)
Fondamentale in ambienti Windows per file sharing.
```bash
# Trovare host con SMB attivo
nmap -v -p 139,445 <network/24>

# Enumerazione via script Nmap
nmap -p 445 --script=smb-os-discovery <target>

# Tool dedicato per NetBIOS
nbtscan -r <network/24>
```

### SMTP (Mail - Porta 25)
Utilizzato per scoprire utenti validi sul sistema tramite comandi testuali.
*   `VRFY <user>`: Verifica se l'utente esiste.
*   `EXPN <list>`: Espande una mailing list.
```bash
# Esempio manuale via Telnet/Netcat
nc -nv <target> 25
VRFY root
```

>[!tip] Consiglio per l'esame
>Se ti viene chiesto di mappare una rete sconosciuta, segui questo ordine:
>1. `nmap -sn` (Host discovery)
>2. `nmap -sS -T4` (Port scan veloce)
>3. `nmap -sV -sC -O` (Approfondimento su versioni e OS degli host "up")
