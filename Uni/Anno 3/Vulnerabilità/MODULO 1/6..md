# Linux Offensive Security – Exploitation
## 1. Finding Attack Vector
### Cos’è
È la fase in cui cerchi **come ottenere privilegi più alti** dopo l’accesso iniziale.
👉 Non è più solo raccogliere info (enumeration), ma:
- trovare **punti deboli sfruttabili**
- scegliere **come attaccare il sistema**

### Differenza con Enumeration
- **Enumeration** → raccolgo info (utenti, servizi, permessi)
- **Attack vector** → uso quelle info per **eseguire un attacco reale**


## 2. Kernel Exploit
### Definizione
Programmi che sfruttano vulnerabilità del kernel per ottenere privilegi root.
### Quando usarli
- se il sistema **non è aggiornato**
- versione kernel vulnerabile
### Tool
- `searchsploit` → cerca exploit già pronti

### Nota importante
⚠️ Non usarli come prima scelta
- possono rompere il sistema
- sono rumorosi
- poco “stealth”


## 3. Dirty COW (esempio)
### Idea
Permette di **modificare file protetti** → escalation a root
### Cosa può fare
- sostituire root
- modificare `/etc/passwd`

### Perché non usarlo subito
- instabile
- facilmente rilevabile

## 6. Path Hijacking
### Idea
Sfrutti come il sistema cerca i comandi

### Come funziona
1. c’è uno script (es. usa `curl`)
2. non hai permessi per eseguirlo direttamente
3. scopri il PATH
4. crei un comando con lo stesso nome
5. lo fai eseguire al posto di quello originale

## 7. Wildcards Exploit

### Problema

👉 puoi manipolare il comportamento dello script


## 8. Sudoers Path Traversal
### Scenario
![[Pasted image 20260325193433.png]]
Puoi usare solo:
```
cat /opt/scripts/*
```

### Problema
Vuoi leggere `/etc/shadow` ma non puoi.

### Bypass
Usi:
```
../
```
👉 vuol dire TORNA ALLA DIRECTORY GENITORE DI QUELLA ATTUALE.

Esempio:
```
/opt/scripts/../../../../etc/shadow
```
👉 da `/opt/scripts` torna indietro di 5 cartelle (che è la `root`) e poi entra in `/etc/shadow`

#### 7z argument injection
- ingannare 7zip per fargli interpretare male i nomi dei file
	- slide 21
		- creo un file `@tosteal` con touch, dentro ci scrivo una stringa con `echo "/etc/passwd" > tosteal`
		- `7za a backup.7z -t7z -snl *`
			- creando il backup interpreta il file `@tosteal` come una riga da eseguire, quindi fa il backup di `shadow`
			- la @ viene usata per definire una lista di file da eseguire

#### Bash script wildcard and input vulnerabilities
- posso usare negli script ad esempio `*`
- per far dare corrispondenza di password, in questo caso `db pass` è uguale a `user pass`
