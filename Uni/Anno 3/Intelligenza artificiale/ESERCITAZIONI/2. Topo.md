3. ==TIPO DI AGENTE==
	- L'agente scelto è a goal che può agire come un agente pianificatore. Dato che l'ambiente è statico e totalmente osservabile può calcolare l'intero piano ottimale offline prima di muoversi.

4. ==Descrizione stato del mondo==
	- **Lo stato del mondo** è la descrizione completa della configurazione attuale dell’ambiente, contenente tutte le informazioni rilevanti e indipendenti dall’agente.
		Formalmente è una N-UPLA: $$S = (POS_{topo}, \ POS_{formaggio}, \ POS_{uscita}, \ \text{Insieme POS buche})$$
	- **Lo stato dello spazio di ricerca** è una descrizione astratta e minimale di una configurazione possibile del mondo, così come usata dall’algoritmo di ricerca (A*, BFS, DFS…).
		Formalmente anche qui è una N-UPLA $$S = (POS_{topo}, \ POS_{formaggio}, \ POS_{uscita}, \ \text{Insieme POS buche})$$
		- **DESCRIZIONE NODO NEL GRAFO** -> i nodi che costituiscono il grafo che è un'astrazione dello spazio di ricerca dell'algoritmo sono strutturati nel seguente modo
			- `n.stato` -> informazioni del singolo stato
			- `n.H` -> valore di $f(n)$ associata al nodo
			- tutti i possibili archi adiacenti al nodo
		
	Quindi in questo contesto è utile differenziarli.
	
	Il flag risulta molto utile per via degli stati goal che l'agente deve raggiungere, infatti 
	- $S_{start}$ = Posizione ingresso (pos iniziale del topo)
	- $S_{goal}$ = Posizione del formaggio / Posizione uscita
		- MA il secondo goal può essere raggiunto solo dopo aver soddisfatto il primo, ecco perché il flag risulta efficace.
	
	
	- ==CARDINALITÀ STATI==
		- È il numero di stati possibili, dato da
			- quanti stati effettivi ho (es. numero di celle)
			- quanti stato aggiuntivi posso avere (es. con un flag binario ho altri due stati) $$|S| = \text{NUMERO DI CELLE TOTALI} \ \times \text{STATI AGGIUNTIVI}$$Nel nostro caso $$|S| = 5 \times 5 \times 2$$
	
	
	- ==Modello di transizione== 
		- **IL TOPO SI MUOVE MA NON HA MANGIATO IL FORMAGGIO** $$a =  move(x' , y')$$ $$Result(s, a) = (NEWPOS_{topo}, \ POS_{formaggio}, \ POS_{uscita}, \ \text{Insieme POS buche}, \ FALSE)$$con
			- SE `move = sx` => $x' \in \{1, ..., 4\}$ e $y \in \{1,..., 5\}$
			- SE `move = dx` => $x' \in \{2, ..., 5\}$ e $y \in \{1,..., 5\}$
			- SE `move = su` => $x' \in \{1, ..., 5\}$ e $y \in \{1,..., 4\}$
			- SE `move = giù` => $x' \in \{1, ..., 5\}$ e $y \in \{2,..., 5\}$
	
	
		- **IL TOPO MANGIA IL FORMAGGIO** $$a = mangia(formaggio)$$ $$Result(s, a) = (POS_{topo}, \ POS_{formaggio}, \ POS_{uscita}, \ \text{Insieme POS buche}, \ TRUE)$$
	
		- **IL TOPO SI MUOVE AVENDO GIÀ MANGIATO IL FORMAGGIO** $$a = move(x', y')$$ $$Result(s, a) = (NEWPOS_{topo}, \ POS_{formaggio}, \ POS_{uscita}, \ \text{Insieme POS buche}, \ TRUE)$$con
			- SE `move = sx` => $x' \in \{1, ..., 4\}$ e $y \in \{1,..., 5\}$
			- SE `move = dx` => $x' \in \{2, ..., 5\}$ e $y \in \{1,..., 5\}$
			- SE `move = su` => $x' \in \{1, ..., 5\}$ e $y \in \{1,..., 4\}$
			- SE `move = giù` => $x' \in \{1, ..., 5\}$ e $y \in \{2,..., 5\}$
	
	
	- ==COSTO AZIONE==
		- dipende dal costo dato dal problema
	
	- ==MISURA DI PRESTAZIONE== 
		- somma di tutti i passi eseguiti CHE VUOLE ESSERE LA MINIMA


- ==FUNZIONE AGENTE==
	- L'agente utilizza come algoritmo di ricerca `A*`, in quanto si tratta di un algoritmo offline che permette di calcolare in maniera ottimale la corretta sequenza di azioni. Successivamente esegue, una ad una, tutte le azioni presenti nel piano generato (in questo caso avremo due piani, uno per ogni obiettivo).
	
	- L'algoritmo `A*` risulta essere
		- COMPLETO -> in quanto trova sempre una soluzione se questa esiste e se vale che $$g(n) \ge d(n) \cdot \epsilon$$ con
			- `g(n)` -> costo del cammino calcolato finora
			- `d(n)` -> profondità del nodo
			- $\epsilon > 0$ -> costo delle azioni
		
		- OTTIMALE -> poiché l'algoritmo espande sempre prima i nodi più promettenti (quindi quelli con una `f(n)` più bassa).
	
	- La funzione di valutazione di `A*` è $$f(n) = g(n) + h(n)$$ dove
		- `g(n)` -> costo del cammino calcolato finora
		- `h(n)` -> stima del costo del cammino per raggiungere l'obiettivo
	
	- L'euristica scelta è la **Manhattan distance**, la quale è
		- AMMISSIBILE -> in quanto non sovrastima mai il costo effettivo per raggiungere l'obiettivo ($h(n) \le h^{*}(n)$)
		- MONOTÒNA -> le stime non decrescono mai ($h(n) \le c(n, n') + h(n')$)
		
		- CALCOLO MANHATTAN DISTANCE: con $x_{1}, y_{1}, x_{2}, y_{2}$ abbiamo $$|x_{1}- x_{2}| + |y_{1} - y_{2}|$$

function TOPO-PROG(percezione) returns azione
	persistent  stato
				piano
				problema
				goal
				FLAG_mangiato = FALSE
	
	
	if FLAG_mangiato == FALSE and POS_topo == POS_formaggio and then
		FLAG_mangiato = TRUE
		svuota piano
		return mangia
	
	
	else if FLAG_mangiato == TRUE and POS_topo == POS_uscita then
		return esci
	
	
	if piano is empty then
		if FLAG_mangiato == FALSE then
			goal = POS_formaggio
		else
			goal = POS_uscita
		
		problema = FORMULATE-PROBLEM(percezione, goal)
		piano = A*(problem)
			if piano == failure then
				return NULL action
	
	
	azione = FIRST(piano)
	piano = rest(piano)
	return azione

- A*
```scss
function A*(problema) returns una soluzione oppure failure
	nodo = nodo con 'stato = problema.stato_iniziale'
	frontieta = coda con priorità ordinata in base a 'f(n)' con inizialmente solo 
                'nodo' come unico nodo
    
    esplorati = insieme di nodi esplorari (inizialmente vuoto)
    
    loop do
	    if frontiera.isEmpty then 
		    return failure
	    
	    nodo = POP(frontiera)
	    
	    if problema.GOALTEST(nodo.stato) then 
		    return SOLUZIONE(nodo)
		    
		aggiungi nodo.stato a 'esplorati'
		
		for each azione in problema.AZIONI(nodo.stato) do
			figlio = CHILD-NODE(problema, nodo, azione)
			if figlio.stato non in frontieta and not in esplorati then
				frontieta = INSERT(figlio.stato)
			else if figlio.stato is in frontieta con f(n) più alto then
				sostituisci quel nodo nella frontiera con "figlio.state" 	
```
