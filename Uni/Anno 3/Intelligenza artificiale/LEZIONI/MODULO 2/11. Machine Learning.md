# Machine Learning
Il machine learning (ML) è un sottoinsieme dell'intelligenza artificiale che si occupa di **creare sistemi in grado di imparare e migliorare autonomamente** man mano che elaborano più dati.

Un agente **impara** da un’esperienza **E**, rispetto a una classe di compiti **T** e a una misura di prestazione **P**, se la sua prestazione su **T**, misurata con **P**, migliora grazie a **E**.

In pratica:
- **E**: dati, esempi, interazioni passate
- **T**: il problema da risolvere (classificare, decidere, prevedere)
- **P**: come misuriamo se sta andando meglio


# Progettare un sistema di apprendimento
Per costruire un sistema di Machine Learning servono quattro scelte chiare:
1. **Training experience**
    - Che tipo di esempi usiamo.
    - Esempio: mosse corrette, scenari migliori, stati osservati.
    
2. **Funzione target (f)**
    - È la funzione che vogliamo imparare.
    - Esempio: data una situazione, quale mossa fare.
    
3. **Rappresentazione dell’ipotesi (h)**
    - Come rappresentiamo la funzione.
    - Esempio: albero di decisione, funzione lineare con pesi.
    
4. **Algoritmo di apprendimento**
    - Il metodo che useremo per trovare un’approssimazione di f.


# Inductive Learning
L’apprendimento induttivo si basa su far apprendere una funzione a partire da esempi dati.
- **f**: funzione target (vera, ma sconosciuta)
- **x**: descrizione di uno stato
- **f(x)**: output corretto per quello stato
- **Esempio**: coppia `(x, f(x))`

## Obiettivo
Trovare un’ipotesi **h** tale che **h ≈ f**, usando un training set di esempi.
Questo modello è semplificato perché:
- ignora la conoscenza di base
- assume che i dati siano disponibili

## Metodo di apprendimento induttivo
Si costruisce o si modifica **h** per farla coincidere con **f** sul training set.
- **Ipotesi consistente**: h(x) = f(x) per tutti gli esempi del training set

### Rasoio di Ockham

Tra più ipotesi consistenti, si preferisce quella **più semplice**.
Idea chiave:
- ipotesi troppo complesse tendono a generalizzare peggio


# Sistema induttivo vs sistema deduttivo
## Sistema induttivo:
  - riceve esempi di training
  - seleziona (implicitamente) un’ipotesi h da uno spazio di ipotesi H
  - il bias induttivo è nascosto nell’algoritmo e nella scelta di H
   - dà l’impressione che il sistema “apprenda dai dati”

### Come funziona il learning nella pratica
- ##### **INPUT**
	Il sistema riceve:
	- **Training examples** → esempi (x, f(x))
	- **New instance** → un nuovo input mai visto
	
- ###### **COSA FA IL SISTEMA**
	- 1️⃣ “Acquire the model (H) through Machine Learning”
		Qui succedono due cose _implicite_:
		- scegli una **classe di ipotesi** H  
		    (es. linee, alberi, reti neurali…)
		- usi i dati per **scegliere una h ∈ H**
			H è **lo spazio delle ipotesi possibili**
		
	- 2️⃣ “Using the model, or Hypothesis Space, H”
		- una volta scelta h la applichi a una nuova istanza
	
- ##### **OUTPUT**:
	- classificazione
	-  oppure “don’t know”

>[!important] IMPORTANTE
Nel sistema induttivo:
>- **H è nascosto**
>- **le assunzioni non sono dichiarate**
>- sembra che il sistema “impari dai dati”
👉 Ma questo è solo **il punto di vista operativo**


# Sistema deduttivo:
- riceve gli stessi esempi di training
- esplicita l’assunzione che la funzione target f (sconosciuta) appartenga allo spazio H
- usa deduzione logica per determinare l’output sul nuovo esempio
- rende esplicite le assunzioni che nel sistema induttivo sono implicite

### Come funziona il learning nella pratica
- ##### **INPUT aggiuntivo fondamentale** 
	- Oltre a:
		- training examples
		- new instance
	- compare una nuova assunzione:
		-   “Assertion: _H contains the target concept_”
		Questa è la frase **più importante di tutto lo schema**.
		Stai dicendo al sistema:
>[!tip] “La vera funzione f che vogliamo imparare  
> **appartiene allo spazio delle ipotesi H**.”
>
>Formalmente:
>$f \in H$
>⚠️ Questa NON è dedotta dai dati.  
>⚠️ Questa è un’**assunzione a priori**.

- ###### **COSA FA ORA IL SISTEMA**
	Non “impara”, ma:
	- usa un **theorem prover** (deduzione logica)
	- verifica:
	    - quali ipotesi in H sono compatibili con gli esempi
	    - cosa segue logicamente per il nuovo caso
	Se:
	- tutte le ipotesi compatibili danno la stessa risposta → classifica
	- altrimenti → “don’t know”


## Learning Decision Trees
### Problema di esempio
Decidere se **aspettare o no un tavolo al ristorante**, usando attributi come:
- presenza di alternative
- fame
- numero di clienti
- prezzo
- tempo di attesa
- prenotazione
- tipo di ristorante

L’output è una decisione: **Sì / No**.

## Alberi di decisione come ipotesi
Un albero di decisione:
- rappresenta una funzione sugli attributi in input
- ha nodi interni = test su attributi
- foglie = decisioni finali

Proprietà importanti:
- esiste un albero coerente per qualsiasi training set
- si preferiscono alberi **compatti**

### Quanti alberi possibili?
Con **n attributi booleani** esistono:
- $2^{2^n}$ alberi di decisione distinti
![[Pasted image 20251212182333.png]]

## Ipotesi congiuntive
Ogni attributo può essere:
- positivo
- negativo
- assente

Numero totale di ipotesi congiuntive:
- $3^n$ 
![[Pasted image 20251212182353.png]]

>[!tip] Nota importante:
>- spazi di ipotesi più espressivi aumentano le possibilità di trovare f    
>- ma aumentano anche le ipotesi compatibili con i dati
>- rischio: peggiorare la generalizzazione


## Decision Tree Learning (DTL)
Obiettivo:
- trovare un albero coerente con il training set

Idea base:
- scegliere ricorsivamente l’attributo **più significativo**
- l’attributo migliore divide i dati in sottoinsiemi il più possibile puri

Schema dell’algoritmo:
- se non ci sono esempi → usa un valore di default
- se tutti gli esempi hanno la stessa classe → ritorna quella classe
- se non ci sono attributi → ritorna la classe più frequente
- altrimenti:
    - scegli l’attributo migliore
    - crea un sottoalbero per ogni valore
![[Pasted image 20251212183316.png]]

## Entropia
L’entropia misura il **disordine** di un training set.
- entropia alta → esempi misti
- entropia bassa → esempi quasi tutti uguali

Serve per capire quanto un attributo è utile a separare i dati.
![[Pasted image 20251212183342.png]]

## Information Gain
Un attributo A divide il training set E in sottoinsiemi ( E_1, …, E_v ).
![[Pasted image 20251212184143.png]]

**Information Gain (IG)**:
- misura la riduzione di entropia dopo la divisione
- indica quanto un attributo migliora la classificazione
![[Pasted image 20251212184314.png]]

Regola:
- scegliere l’attributo con **IG massimo**
